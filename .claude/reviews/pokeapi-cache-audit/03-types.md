# 03 — Types
**Endpoints:** type (43 — sampled 10)
**Schema files touched:** types.ts

## type

- Files sampled: normal (1), fighting (2), poison (4), electric (13), fire (10), dark (17), steel (9), fairy (18), unknown/10001 (past), shadow/10002 (special)
- JSON top-level fields:
  - `id` (int), `name` (string), `damage_relations` (object with 6 nested arrays: `double_damage_from`, `double_damage_to`, `half_damage_from`, `half_damage_to`, `no_damage_from`, `no_damage_to`)
  - `past_damage_relations` (array, per-generation: `{generation, damage_relations}`)
  - `game_indices` (array of `{game_index, generation}`)
  - `generation` (single `{name, url}`)
  - `move_damage_class` (`{name, url} | null`) — physical/special pre-Gen-4
  - `names` (array of `{name, language}`)
  - `pokemon` (array of `{slot, pokemon}` — pokemon with this type)
  - `moves` (array of `{name, url}` — moves of this type)
  - `sprites` (object — per-generation, per-game-version type icon URLs)
- DB tables: `typesTable`, `typeNamesTable`, `typeEfficacyTable`, `typeGameIndicesTable` ([types.ts](../../../database/schemas/pokeapi/types.ts))
- Persisted columns ↔ JSON: standard id/name/url to typesTable; `damage_relations` flattened into typeEfficacyTable rows (one per pairing); `game_indices` → typeGameIndicesTable; `names` → typeNamesTable; `generation.url → generationId` FK; `move_damage_class.url → moveDamageClassId` FK
- **Dropped JSON fields:**
  - `past_damage_relations` (array per-generation) — **arguably useful** (historical type-chart changes, e.g. ghost vs psychic in Gen 1, dark/steel introductions). Currently only the current-generation efficacy persists.
  - `sprites` object (per-game type icon URLs) — **safe to skip** unless UI shows in-game type badges; if needed, add a `typeSpritesTable` mirroring `pokemonSpritesTable`.
  - `pokemon` array — **reconstructable** via pokemonTypesTable join
  - `moves` array — **reconstructable** via moves.type FK
- **DB columns without JSON source (custom — preserve):** none
- Seed handling: standard — find type insert section in [seed.ts](../../../database/seeds/pokeapi/seed.ts). typeEfficacyTable rows generated by walking each of the 6 damage_relations sub-arrays.
- Issues / questions:
  - **past_damage_relations missing** is the only meaningful gap. Would need a `typeEfficacyHistoryTable` with `(generationId, typeId, vsTypeId, multiplier)` to capture this.
  - Type icon sprites: confirm UI doesn't need them (the app's `pokemonSpritesTable` is for pokemon, not types).

None.

## Group-level observations

1. **`past_damage_relations` parallels `pokemon-species.past_types` and `pokemon.past_abilities/past_stats`** — pattern of "historical override per generation" appears in multiple endpoints. Could justify a shared `*History` table convention.
2. Type sprites are currently dropped — verify intentional. If types ever appear as visual badges in UI, a sprite table is needed.
